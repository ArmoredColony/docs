# Pixie Pivot Changelog

## 1.2.0
03/March/2026

### Added
- Support for targeting mesh elements while in Object Mode.
- Snap/align between multiple objects (still restricted to mesh objects).
- **Lock Cursor Location** → Prevents the 3D cursor from changing location while still allowing it to rotate/align in place.

### Changed
- Converted the add-on into an extension.

### Removed
- **Anchor to Selection** → Too much dev time; may re-add if the API exposes `transform_orientation` and `transform_pivot_point` results. Replaced with simpler **Lock Cursor Location**.

## 1.1.4
09/February/2026

### Changed
- Setting the Origin no longer forces GLOBAL Orientation and MEDIAN_POINT Pivot; it keeps previous settings.

## 1.1.3
11/October/2025

### Fixed
- Error when enabling `anchor_to_selection` without a selection.

## 1.1.2
06/October/2025

### Added
- Behavior Preference **Double Tap Resets To** with pivot reset options.

### Changed
- Pressing 'DD' now resets the pivot to the previous non-cursor settings instead of always resetting to GLOBAL Orientation and MEDIAN_POINT pivot.

## 1.1.1
06/September/2025

### Fixed
- AttributeError caused by incorrect import of `view3d_utils` from `bpy_extras`.

## 1.1.0
03/September/2025

### Added
- Toggle Key 'Anchor to Selection' [T]: Allows alignment to other elements while keeping pivot near selection.

### Fixed
- HUD position bug when 3D region was not at bottom left.
- Update availability false positive when addon crashed.

### Changed
- HUD drawing restricted to single 3DView.
- Mouse HUD slightly enlarged.

## 1.0.0
28/August/2025

### Initial Release
